# Forgum — Prompt Integration (Shell Hooks that Don't Break the Prompt)

> **The contract:** after `forgum init <shell>`, the user's prompt behaves *exactly* as before — same prompt string, same keybindings, same history, same completions — *plus* `forgum` renders the right cow, with the right eyes, the right effect, and the right lolcat setting, read from the live config. And a background daemon (if running) is swept clean if it dies.

The current shell-hook layer breaks this contract in **six** ways (BUG-S1–S8). This document is the redesign.

---

## 1. What's broken today (recap)

| Bug | Symptom |
|-----|---------|
| BUG-S1 | bash/zsh hooks emit **invalid JSON** (backslashes not escaped) → engine errors, no animation |
| BUG-S2 | fish hook: `'\n'` is literal, not newline → invalid JSON |
| BUG-S3 | hooks read `$conf.effect` / grep `"effect"` → config key is `animation.mode` → always `random` |
| BUG-S4 | hooks call **system `cowsay`** → no Forgum cows/eyes/lolcat; fails if cowsay absent |
| BUG-S5 | `forgum-engine` not on PATH → `command not found` |
| BUG-S6 | two generators (PowerShell + Rust) emit **different** hooks |
| BUG-S7 | effect baked at `init` time → config changes ignored |
| BUG-S8 | WSL/Git Bash misdetected as `pwsh` |
| BUG-B1 | background engine **steals keystrokes** → Enter exits the animation |

Fixes are below, grouped by concern.

---

## 2. Architecture: one generator, engine-native rendering

```
                    ┌─────────────────────────────────────┐
   forgum init bash │  forgum-engine init bash            │
   ───────────────► │  (single source of truth, in Rust)  │
                    │  emits a bash hook that:            │
                    │   1. resolves engine path (baked)   │
                    │   2. builds JSON via a real encoder │
                    │   3. calls engine with cow_file +   │
                    │      text + config path             │
                    └──────────────┬──────────────────────┘
                                   │ stdout = hook script
                                   ▼
              user pastes into ~/.bashrc ──► `forgum "hello"` works
```

**Decisions:**

1. **`handle_init` in Rust is the only generator.** `GetForgumShellHook.ps1` is deleted (or reduced to `forgum-engine init $shell`). Kills BUG-S6.
2. **The engine renders cows natively.** `forgum-engine` gains a `cow_file` field + a bundled cow directory. It expands `$eyes/$tongue/$thoughts`, wraps the fortune in a speech bubble, and renders. The hook no longer calls system `cowsay`. Kills BUG-S4.
3. **The engine reads the config itself.** A new `--config <path>` flag (default: the platform config path) makes the engine the single reader of `config.json`. The hook never parses JSON. Kills BUG-S3, BUG-S7.
4. **JSON is built by the engine's own `serde_json`, not string substitution.** The hook passes `text`/`cow_file`/`eyes` as **separate argv flags**, and the engine assembles the `SceneConfig` internally. Kills BUG-S1, BUG-S2.

## 3. The engine CLI (replaces JSON-on-stdin for interactive use)

Interactive use switches from "pipe JSON" to "argv flags." The JSON-on-stdin path is kept for programmatic/PowerShell use and for `--file`.

```
forgum-engine render \
  --cow default \
  --eyes oo --tongue "  " \
  --text "Hello, world!" \
  --effect auto \
  --background \
  --duration 0 \
  --config ~/.config/Forgum/config.json
```

The engine:
1. parses argv,
2. reads `config.json` (the `--config` path, or platform default),
3. merges: argv > config > built-in defaults,
4. loads `Data/Cows/<cow>.cow`, expands placeholders, wraps `--text` in a bubble → `cow_text`,
5. resolves `effect` (`auto` → `style_matcher` based on `cow_file`),
6. builds `SceneConfig` and enters `render_loop_background` / `render_loop_foreground`.

**The hook is now trivial and shell-portable** — it just calls this CLI with the user's args. No JSON, no grep, no cowsay.

## 4. The bash hook (final form)

Generated by `forgum-engine init bash`. Path to the engine is **baked in** at generation time (BUG-S5).

```bash
# >>> forgum (bash) >>>
# Generated by forgum-engine init bash  (do not edit by hand)
__FORGUM_ENGINE="/usr/local/bin/forgum-engine"
__FORGUM_CONFIG="${FORGUM_CONFIG:-$HOME/.config/Forgum/config.json}"
__FORGUM_RUNTIME="${XDG_RUNTIME_DIR:-/tmp/forgum-$UID}/forgum"
mkdir -p "$__FORGUM_RUNTIME"

forgum() {
  local text="$*"
  if [ -z "$text" ]; then
    text="$("$__FORGUM_ENGINE" fortune --config "$__FORGUM_CONFIG" 2>/dev/null)"
  fi
  "$__FORGUM_ENGINE" render \
    --config "$__FORGUM_CONFIG" \
    --text "$text" \
    --background --duration 0 "$@"
}

# Sweep a dead background daemon so the prompt stays clean (§6 of 03-…)
__forgum_precmd() {
  local state="$__FORGUM_RUNTIME/daemon.json"
  if [ -f "$state" ]; then
    local pid; pid=$(command jq -r .pid "$state" 2>/dev/null || awk -F\" '/pid/{print $4}' "$state")
    if [ -n "$pid" ] && ! kill -0 "$pid" 2>/dev/null; then
      local rows cols; rows=$(awk -F\" '/ob_y1/{print $4}' "$state"); cols=${COLUMNS:-80}
      printf '\x1b7'
      local y=1; while [ "$y" -le "$rows" ]; do printf '\x1b[%d;1H%*s' "$y" "$cols" ''; y=$((y+1)); done
      printf '\x1b8\x1b[0m'
      rm -f "$state"
    fi
  fi
}
PROMPT_COMMAND="__forgum_precmd${PROMPT_COMMAND:+; $PROMPT_COMMAND}"
# <<< forgum <<<
```

**Why this is correct:**
- No JSON hand-rolling — `--text` is an argv flag; `serde_json`/clap handle encoding inside the engine.
- `--config` lets the engine read the live `animation.mode` (BUG-S3/S7 fixed).
- `forgum fortune` is a new engine subcommand that picks a random fortune from `Data/Fortunes/fortunes.txt` (+ user db) — no PowerShell needed.
- The engine path is absolute → no PATH dependency (BUG-S5).
- `PROMPT_COMMAND` sweep handles `kill -9` (the §9.3 net from the engine doc).
- Only bash 3.2-compatible syntax (no `mapfile`, no `${var,,}`) → works on macOS's licensed bash.

### 4.1 Passing cow/eyes/effect through

```bash
forgum --cow dragon --eyes '@@' --effect fire "burn stuff"
```
Because `forgum()` forwards `"$@"` after the fixed flags, and clap parses known flags, this Just Works. Unknown flags error with a helpful message (BUG-P15 — no more silent drops).

## 5. The zsh hook

Identical to bash except:
- `PROMPT_COMMAND` → `precmd_functions+=(__forgum_precmd)` (zsh doesn't use `PROMPT_COMMAND`).
- The sweep function is the same.
- Completions: `_forgum` is regenerated from `forgum-engine --list-effects` (BUG-I3).

```zsh
# >>> forgum (zsh) >>>
__FORGUM_ENGINE="/usr/local/bin/forgum-engine"

forgum() {
  local text="$*"

  "$__FORGUM_ENGINE" render --config "$__FORGUM_CONFIG" --text "$text" --background --duration 0 "$@"
}

__forgum_precmd() {
  # … same sweep logic as bash …
}
precmd_functions+=(__forgum_precmd)
# <<< forgum <<<
```

## 6. The fish hook

```fish
# >>> forgum (fish) >>>
set -g __forgum_engine "/usr/local/bin/forgum-engine"
set -g __forgum_config "$FORGUM_CONFIG" ; or set -g __forgum_config "$HOME/.config/Forgum/config.json"
set -g __forgum_runtime (set -q XDG_RUNTIME_DIR; and echo "$XDG_RUNTIME_DIR/forgum"; or echo "/tmp/forgum-"(id -u)"/forgum")
mkdir -p $__forgum_runtime

function forgum
    set text $argv
    if test (count $argv) -eq 0
        set text ($__forgum_engine fortune --config $__forgum_config 2>/dev/null)
    end
    $__forgum_engine render --config $__forgum_config --text "$text" --background --duration 0 $argv
end

function __forgum_sweep --on-event fish_prompt
    set state $__forgum_runtime/daemon.json
    if test -f $state
        set pid (jq -r .pid $state 2>/dev/null)
        if test -n "$pid"; and not kill -0 $pid 2>/dev/null
            set rows (jq -r .ob_y1 $state); set cols $COLUMNS
            printf '\x1b7'
            for y in (seq 1 $rows); printf '\x1b[%d;1H%*s' $y $cols ''; end
            printf '\x1b8\x1b[0m'
            rm -f $state
        end
    end
end
# <<< forgum <<<
```
No `'\n'`-as-literal bug (BUG-S2) because we never build JSON in fish — all encoding is in the engine.

## 7. The PowerShell hook

PowerShell already works (it uses `ConvertTo-Json`), but it must switch to the same `render` CLI for consistency:

```powershell
# >>> forgum (pwsh) >>>
$__ForgumEngine = Get-EngineBinary
$__ForgumConfig = Get-ConfigPath

function forgum {
    param([Parameter(ValueFromRemainingArguments=$true)][string[]]$Args)
    $text = if ($Args) { ($Args -join ' ') } else { & $__ForgumEngine fortune --config $__ForgumConfig }
    & $__ForgumEngine render --config $__ForgumConfig --text $text --background --duration 0 @Args
}

# Sweep on each prompt
$global:__ForgumPromptBackup = $function:prompt
function global:prompt {
    $state = Join-Path $env:TEMP "Forgum\daemon.json"
    if (Test-Path $state) {
        $info = Get-Content $state -Raw | ConvertFrom-Json
        if ($info.pid -and -not (Get-Process -Id $info.pid -EA SilentlyContinue)) {
            [Console]::Out.Write("`e7")
            1..$info.ob_y1 | % { [Console]::Out.Write("`e[$($_);1H$(' '*($Host.UI.RawUI.WindowSize.Width)))") }
            [Console]::Out.Write("`e8`e[0m")
            Remove-Item $state -Force
        }
    }
    & $global:__ForgumPromptBackup
}
# <<< forgum <<<
```

## 8. Shell detection (fixing BUG-S8)

`forgum-engine init` with no shell argument auto-detects:

```rust
fn detect_shell() -> ShellKind {
    if let Ok(s) = env::var("FORGUM_SHELL") { return ShellKind::parse(&s); }
    // parent process name — works inside WSL, Git Bash, Cygwin, MSYS2
    if let Some(name) = parent_comm() {  // /proc/$PPID/comm (Linux), ps -o comm= (macOS), …
        match name.as_str() {
            "bash" | "-bash" => return Bash,
            "zsh"  => return Zsh,
            "fish" => return Fish,
            "pwsh" | "powershell" | "powershell.exe" => return Pwsh,
            _ => {}
        }
    }
    if let Ok(s) = env::var("SHELL") { return ShellKind::parse(&s); }
    if cfg!(windows) { Pwsh } else { Bash }
}
```
`parent_comm()` reads `/proc/$PPID/comm` on Linux, runs `ps -o comm= -p $PPID` on macOS/BSD, and uses WMI `Win32_Process` on Windows. This correctly returns `bash` for a Git Bash user on Windows (BUG-S8 fixed).

## 9. Completions (regenerated, never stale)

`forgum-engine --list-effects` prints the canonical effect list. A `forgum-engine completions <shell>` subcommand emits the completion script using `clap_complete`. CI asserts the committed completion scripts match `forgum-engine completions` — kills BUG-I3 permanently.

```bash
# in CI:
forgum-engine completions bash   > scripts/completions/forgum.bash
forgum-engine completions zsh    > scripts/completions/_forgum
forgum-engine completions fish   > scripts/completions/forgum.fish
git diff --exit-code scripts/completions/   # fails if drifted
```

## 10. The prompt-safety checklist (reviewer gate)

Every change to a shell hook must pass:

- [ ] Hook contains **zero** JSON string concatenation. All encoding is `serde_json` inside the engine.
- [ ] Hook contains **zero** `cowsay`/`fortune`/`lolcat` external calls. Everything is `forgum-engine`.
- [ ] Engine path is absolute (baked at `init` time) or resolved via a `forgum-engine` on PATH that the installer guarantees.
- [ ] `precmd`/`PROMPT_COMMAND`/`fish_prompt` sweep is present and idempotent.
- [ ] Hook works on bash 3.2 (macOS) — `shellcheck --shell=bash` clean.
- [ ] Hook works on fish 3.x — `fish --no-execute` parses without error.
- [ ] Hook never calls `exec` or replaces the shell process.
- [ ] Hook never modifies `PS1`/`PROMPT`/`prompt` beyond appending the sweep.
- [ ] Typing `forgum hello<Enter>` runs the animation; the **same** `<Enter>` also submits future shell commands (no keystroke theft — verified by the pty test in `03-…` §13).
