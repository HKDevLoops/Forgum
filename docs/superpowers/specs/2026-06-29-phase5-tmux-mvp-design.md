# Phase 5 — tmux MVP Design Spec

> Multiplexer detection, per-pane daemon state, tmux install, status line, and popup cow.

---

## Goal

Make Forgum work inside tmux: detect the multiplexer, key daemon state on pane ID, generate tmux config, provide a status-line one-shot, and support popup-mode rendering.

---

## Deliverables

| # | Feature | Description |
|---|---------|-------------|
| M1 | `detect_mux()` | Platform function to detect tmux/zellij/screen/wezterm from env vars |
| M2 | Per-pane state | Daemon state files keyed on `$TMUX_PANE` instead of shell PID |
| M3 | `forgum tmux install` | CLI subcommand that writes tmux config block to stdout |
| M4 | `status-line` | One-shot engine mode that prints a single-line fortune+cow for tmux status bar |
| M5 | `--popup` mode | Foreground render inside tmux `display-popup` |

## M1: Multiplexer Detection

Add to `forgum-platform/src/paths.rs` (or a new `mux.rs` module):

```rust
pub enum Mux {
    None,
    Tmux { pane: String, session: String },
    Zellij { tab: String },
    Screen { window: String },
    WezTerm { tab: String },
}

pub fn detect_mux() -> Mux {
    if let Ok(v) = std::env::var("TMUX") {
        let pane = std::env::var("TMUX_PANE").unwrap_or_default();
        // Parse session from TMUX env: "/tmp/tmux-1000/default,1234,0"
        let session = v.split(',').nth(1).unwrap_or("").to_string();
        return Mux::Tmux { pane, session };
    }
    if std::env::var("ZELLIJ").is_ok() {
        return Mux::Zellij {
            tab: std::env::var("ZELLIJ_TAB_NAME").unwrap_or_default(),
        };
    }
    if std::env::var("STY").is_ok() {
        return Mux::Screen {
            window: std::env::var("WINDOW").unwrap_or_default(),
        };
    }
    if std::env::var("WEZTERM_UNIX_SOCKET").is_ok() {
        return Mux::WezTerm {
            tab: std::env::var("WEZTERM_TAB_ID").unwrap_or_default(),
        };
    }
    Mux::None
}
```

Export `detect_mux` and `Mux` from `lib.rs`.

## M3: `forgum tmux install` Subcommand

Add a `Tmux` subcommand to the CLI:

```
forgum tmux install    # prints tmux config block to stdout
```

The generated config:

```bash
# >>> forgum tmux >>>
# Generated by forgum-engine tmux install — do not edit by hand
set -g status-right-length 100
set -g status-interval 5
set -g status-right "#(forgum-engine status-line --max-len 70 2>/dev/null)"
set-hook -g pane-focus-in  'run-shell "forgum-engine --background --daemon 2>/dev/null"'
set-hook -g pane-focus-out 'run-shell "forgum-engine control send stop 2>/dev/null"'
# <<< forgum tmux <<<
```

## M4: Status Line One-Shot

Add a `status-line` subcommand to the engine CLI:

```
forgum-engine status-line [--max-len N] [--format tmux]
```

Behavior:
1. Read config (cow, effect, eyes, etc.)
2. Pick a random fortune (or cache per invocation)
3. Render a single-frame mini-cow + fortune
4. Print as single line (ANSI truecolor for tmux)
5. Exit immediately (no animation)

The output is a single line with ANSI color codes that tmux passes through to the status bar.

---

## M5: Popup Mode (`--popup`)

Add `--popup` flag to the CLI. When set:
1. Print tmux `display-popup` wrapper sequence
2. Run foreground render loop inside the popup
3. On exit, popup closes automatically

This is the "summon a cow in a floating overlay" feature.
