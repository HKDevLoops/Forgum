fix: resolve cross-platform code defects behind CI failures

- Tests/Forgum.PesterHelpers.psm1: stop hardcoding forgum-engine.exe on
  all platforms. Branch on $IsWindows so Linux/macOS resolve the extensionless
  binary; fixes every engine-invoking Pester test on non-Windows.

- crates/engine/tests/visual_render.rs: golden regression now keys the
  blake3 hash per target OS (feature_tag includes std::env::consts::OS).
  Pixel frames are not bit-identical across platforms (f32 math, sin/cos,
  -0.0 vs 0.0), so the previous cross-OS comparison falsely failed (e.g. the
  "eyes" cow on Windows). Each OS self-golds and still catches real
  regressions within that platform.

- crates/engine/tests/daemon_lifecycle.rs: replace brittle 1s fixed sleeps
  and fragile PID parse with polling loops (10s deadlines) for socket-bind
  and process-exit. Robust under CI load; removes the flaky exit-101.

- crates/engine/tests/windows_specific.rs: drop the `duration > 0.001`
  lower bound in timer_runs_cmd_echo; on fast machines Instant can read ~0
  for a sub-millisecond spawn, causing a false failure. Keep the upper bound
  as a hang guard.
